Help — How is sensitive data detected in logs?

Detection reuses the exact same pattern-matching + checksum validation as ticket upload scanning (Luhn for cards, mod-97 for IBAN) — no LLM, no external API call. The file is read in fixed-size chunks and scanned line by line, never loaded whole into memory, so a 100GB+ log scans the same way a small one does, just slower.

Phone number detection is off by default per source — it's the single most expensive check (real parsing, not just regex) and most log lines contain digits that would trigger it constantly. Turn it on per source if you need that coverage.

Only a masked preview is ever stored or shown — never the raw matched value.
