Help — Using this page

Use this page to sanity-check data quality before running clustering — it's a raw look at what actually landed in the database, not a modeled output.

  • Ticket Volume Trend — toggle Day/Week/Month/Quarter to see how ticket volume moves over time at different resolutions.
  • By Country / Status / Priority / Application — straight counts. A value of "Unspecified" or "Unknown" means that field wasn't mapped (or was empty) for those tickets.
  • By Queue / Requested Type — same straight-count treatment. Queue is the field the Categorize Tickets k-NN suggestion feature exists to fill in — a shrinking "Unspecified" bar here reflects that feature's progress on this project.
  • By Offering — only shown once at least one ticket has an offering value. A ticket tagged with more than one offering (e.g. "SunSystems Support, QA Vision") counts toward every offering it lists, not just the first — so bars can sum to more than the total ticket count.
  • Missing Country/App, Missing Queue/Requested Type, and Missing Description KPIs — how many tickets fell back to "Unspecified" (or an empty description) for these fields. High numbers here are worth fixing in Map Columns before clustering, since several of these fields feed downstream summaries or the clustering text itself (see Clustering Settings).

Found a data quality issue? Go back to Upload Data and re-map the affected column, or use Delete on a bad upload from the Recent Uploads table.
