Help — Uploading data

What to upload

A CSV or Excel export of Incidents, Requests, Changes, or Production Support tickets — .csv, .xlsx, or .xls, up to 25 MB. Any column layout is fine; you'll match your columns to fields on the next screen.

The 3-step wizard

  1. Upload File — just the raw file, nothing is read yet.
  2. Map Columns — tell it which of your columns is the title, date, country, etc. Nothing is mandatory; unmapped fields get a sensible fallback instead of blocking the import.
  3. Review & Import — a preview of the first 20 mapped rows, then a full commit of every row.

Recent Uploads

Every commit is versioned (Upload v1, v2, …) so you always know which file produced which tickets. Delete on a row removes only that upload's tickets — re-run your pipelines afterward since existing clusters won't automatically drop the deleted tickets.

Danger Zone — Reset Data

This wipes every ticket, cluster, and upload record for the current project only — other projects are untouched. It's irreversible, which is why it requires typing the project name to confirm. Use it when starting a clean pilot, not for removing one bad upload (use Delete on that row instead).
