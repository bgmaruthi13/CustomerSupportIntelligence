Help — Mapping columns

Nothing is required

Tick Include and pick a source column for whichever fields you want. Anything left unmapped (or that fails to parse) gets a sensible auto-generated fallback at import time — it never blocks the row. Fields tagged RECOMMENDED aren't mandatory either, they just matter most for clustering quality: a good title and date make far better clusters than a perfectly complete row of metadata.

Use Recommended Fields Only to quickly strip the mapping down to just those, if you want a fast first pass.

Date Format

Applied to whatever column you map to "created_at". "Auto-detect" handles most common formats, but pick an explicit one for ambiguous dates like 01/02/2026 (is that 1 Feb or 2 Jan?) to avoid silent misparsing.

Preview

Shows the first 20 rows exactly as they'll be imported. Red text means that value failed to parse and will get a fallback — check a few of these before committing if the count looks high. The preview updates automatically whenever you change a mapping or the date format.

Import

Refresh Preview only re-renders the preview table — nothing is saved. Import N Rows commits the entire file, not just the previewed 20. The only thing that skips a row outright is a genuine duplicate external ID; everything else is imported with a note in the results.
