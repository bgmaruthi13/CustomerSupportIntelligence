Help — How is sensitive data detected?

Every column of each uploaded file is scanned — not just the columns you chose to map — since an unmapped column is exactly the data nobody chose to import and therefore nobody's watching. Detection is pattern matching and checksum validation (Luhn for cards, mod-97 for IBAN), not AI — no LLM, no external API call.

Context is what separates a real leak from noise: a name or email in a column mapped to "Reported By"/"Assigned To" is Expected Identity — that's what the field is for, not a surprise. The same pattern in an Unmapped Column (data nobody chose to import) or a Free Text field (like description) is worth a closer look — those sort first by default.

High confidence (email, IBAN, card) passed a real mathematical checksum. Medium (phone) used a well-tested parsing library but still guesses a region for numbers without a country code. Low (bank account number, address) has no universal format at all and is best-effort — expect some false positives, especially account numbers, which can't be reliably distinguished from ticket/order IDs by pattern alone.

Only a masked preview is ever stored or shown — never the raw matched value.
