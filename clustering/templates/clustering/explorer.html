Help — Reading this view
  • Each point is one ticket. Points sharing a color belong to the same cluster; hover any point to see its ticket ID, title, and cluster name.
  • Grey points are unclustered noise — tickets HDBSCAN couldn't confidently group with anything else.
  • 2D / 3D — both are the same underlying clustering, just projected down to 2 or 3 dimensions (via UMAP) purely for visualization. Position here is cosmetic and never feeds back into the actual clustering or confidence scoring.
  • Switch between Traditional ML and Generative AI at the top to compare how each engine grouped the same tickets — they're computed independently, so the same ticket can land in different-looking neighborhoods on each.
  • On a large dataset, this view samples a subset of points so the browser doesn't choke — cluster counts and stats everywhere else (Problem Clusters, Dashboard) always use the full dataset regardless of what's sampled here.
