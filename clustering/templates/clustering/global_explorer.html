Help — Reading this view
  • Each point is one ticket from the last global run. Position (via UMAP) is cosmetic — it never feeds back into clustering or confidence scoring.
  • Color: By Cluster — same as the per-project Cluster Explorer: points sharing a color belong to the same cluster.
  • Color: By Project — points are colored by which project the ticket came from instead. Two colors occupying the same neighborhood on the map is the visual signature of a cross-project intersection: the same kind of issue, worded similarly, coming from more than one team.
  • Highlight cross-project intersections — boosts the size and opacity of every point that belongs to a cluster with 2+ projects (dims everything else), regardless of which color mode is active, so those points are easy to pick out.
  • Hover any point for its ticket ID, title, cluster, and project.
