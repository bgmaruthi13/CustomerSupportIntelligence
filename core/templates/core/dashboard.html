Help — Getting started on this page

This is the home base for {{ project.name }} — everything here is scoped to whichever project is selected in the top-right switcher.

Typical workflow

  1. Upload Data — bring in a CSV/Excel export from your ITSM tool.
  2. Run Traditional ML and/or Generative AI — both are safe to run on the same data; results are kept separately per engine.
  3. Review Problem Clusters — recurring patterns are ranked by recurrence, with a Candidate ribbon on the ones that clear this run's thresholds (see the Help section on the Problem Clusters page for exactly how that's decided).
  4. Open a cluster's detail page to see its trend, confidence, and member tickets before deciding whether to raise a real Problem Record.

Reading the KPI cards

  • Tickets in Scope — total tickets currently in this project, regardless of which upload they came from.
  • Traditional ML / Generative AI Clusters — how many non-noise clusters each engine currently has saved. These are independent — running one doesn't affect the other.
  • Problem Candidates — count across both engines combined.

No results yet? Nothing here is fabricated — an empty state just means that engine hasn't been run on this project's current data.
